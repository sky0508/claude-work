### 思考・作業スタイル

- **ボトルネック思考**: 「何が詰まっているか」を先に特定してから動く
- **構造化が得意**: 問題を層に分解して独立した打ち手を設計する
- **PDCAを回したい**: 数値→分析→戦略の流れを自動化・型化するのが目標
- **3役割並走**: タスクが散らかりやすい環境。リソース制約を常に意識
- **優先順位**: ①中長期的に重要なこと → ②締め切りが近いもの

### コミュニケーション好み

- 長い説明より**表・箇条書き・構造化された短い情報**が好き
- 回答は「結論 → 根拠 → 打ち手」の順で
- 推奨を積極的に出す（選択肢の羅列より「これが良い」を先に）
- 「はじめまして」から始めない → 文脈を持った対話をする

### AIへの要求

| 自律でやってほしいこと | 必ず確認を取ること |
|----------------------|------------------|
| ファイル・DB作成・更新 | メールの送信 |
| 分析・結論の提示 | カレンダーへの予定追加 |
| 次のアクション提案 | プロジェクトの方向性変更 |
| タスクの優先度判定・自動分類 | 既存データの大幅な書き換え |

## 2. 全体アーキテクチャ

```
【Notion】← 情報の全マスター
  タスクDB / プロジェクトDB / KPI DB / ログDB / メモDB
       ↑↓ MCP                    ↑↓ MCP / API
【Claude Code】              【Base44 + LINE】
  PC作業時の実行エンジン         モバイル秘書（日常の細々した作業）
  実装・分析・DB更新・資料作成     タスク追加・完了・カレンダー・メール
       ↑↓
【Notion AI】
  ドキュメント作成・軽い分析
```

**設計思想**:
- **Notionに全情報を集約**。どのツールからも同じデータを見る
- **Claude Code**はNotionのコンテキストを読んで重い作業を実行し、結果をNotionに書き戻す
- **Base44 + LINE**はスマホから日常的な秘書業務をこなす（Notionを読み書き）
- **ローカルに残すのは最小限**（Claude Codeの行動ルール・スキル定義のみ）

## 4. タスク管理システム

### 4-1. タスクの正データ（2ファイル体制）

| ファイル | 対象 | 内容 |
|---------|------|------|
| `company_jva/secretary/backlog.md` | JVA業務タスク | 企業アウトリーチ・学生管理・KPI等 |
| `company_personal/tasks/backlog.md` | 個人・横断タスク | スキル構築・環境整備・個人目標等 |

### 4-2. 4象限（アイゼンハワーマトリックス）

```
             緊急          非緊急
重要   │ Q1（今日〜2日）│ Q2（今週〜来週）│
非重要 │ Q3（素早く片付け）│ Q4（後回し）  │
```

| 象限 | 説明 | 着手目安 |
|------|------|---------|
| Q1 | 緊急×重要 | 今日〜2日以内 |
| Q2 | 非緊急×重要 | 今週〜来週 |
| Q3 | 緊急×非重要 | 素早く片付けるか委任 |
| Q4 | 非緊急×非重要 | 後回し・削除検討 |

### 4-3. アンビエントタスク検知ルール（常時有効）

会話中にタスクへの言及を検知したら**確認なしで即座に**バックログを更新する。

| 発言パターン | 対応 |
|------------|------|
| 「〇〇やった」「完了した」「終わった」 | 該当タスクを完了マーク `[x]` |
| 「〇〇やらなくていい」「キャンセル」「はいい」 | バックログから削除 |
| 「〇〇やる必要がある」「追加して」「しなきゃ」 | 適切な象限に追加 |
| 「〇〇は来週でいい」「急がない」 | 象限をQ1→Q2等に変更 |

更新後は一言通知のみ:
```
（バックログ更新: [タスク名] を[完了/削除/追加/Q変更]）
```

## 5. スキル一覧

Claude Codeでは「スキル」と呼ばれる自動実行フローがある。
Notionではこれを「AIスキル」として再現する。

### 5-1. スキル一覧と概要

| スキル名 | トリガー | 機能概要 |
|---------|---------|---------|
| **morning** | `/morning`・「今日のブリーフィング」 | Google Calendar・Gmail・バックログを統合して今日を設計。`tasks/today.md`を生成 |
| **ohayo** | 「おはよう」など朝の挨拶全般 | カレンダー取得・学生/企業新着チェック・優先タスク提案・日次TODOファイル作成 |
| **evening** | 「お疲れ」「今日終わり」など | タスク進捗ヒアリング・バックログ更新・翌朝ブリーフィングの精度を上げる |
| **secretary** | 「秘書」「タスク追加」「進捗確認」など | 日中の全タスク管理・カレンダー確認・メール対応・日程調整・壁打ちを担当 |
| **fin_s** | `/fin_s` | セッション終了時に作業内容を自動仕分け保存。ナレッジを適切な場所に蓄積 |
| **weekly_kpi** | `/weekly_kpi` + レポートペースト | Manus AI週次レポートを受け取りKDBを更新、戦略アクションアイテムを生成 |
| **weekly_review** | `/weekly_review` | 今週の成果・未完了・来週の優先事項をまとめてレポートを生成 |
| **weekly_strategy** | `/weekly_strategy` + レポートペースト | KDB反映→KPIギャップ分析→次アクション設計→日本語週次戦略レポート生成 |
| **manus-outline** | 「Manusでスライド作って」など | スライドアウトラインをManus向けに生成。対話→論点深掘り→アウトライン→プロンプト出力 |
| **slides** | 「スライドを作って」など | Googleスライド（PPTX）を自動生成 |
| **cv_send** | `/cv_send` | 学生のCVを企業に送付する秘書業務 |

### 5-2. 主要スキルの詳細フロー

#### morning / ohayo スキル

```
1. Google Calendarから今日・今週の予定を取得（MCP）
2. 学生スプレッドシートから新規応募を検出（GAS）
3. 企業スプレッドシートから新規申請を検出（GAS）
4. バックログ・前回セッションログを読み込む
5. 今日のTODOファイル（YYYY-MM-DD.md）を生成
6. 優先タスクTop3を提示してブリーフィング表示
```

出力先: `company_jva/secretary/todos/YYYY-MM-DD.md` + `tasks/today.md`

#### secretary スキル

```
発言パターン判定 → モード選択
  タスク追加 → 象限自動判定 → バックログ更新（確認なし）
  タスク完了 → 即マーク（確認なし）
  進捗確認  → Q1タスク一覧を優先表示
  相談・壁打ち → コンテキスト読み込み → 結論先出しで回答
  カレンダー → MCPで取得・表示
  メール確認 → 未返信5件抽出・サマリー表示
  日程調整  → 空き時間候補3件提示 → 確認後カレンダー追加
```

**確認が必要なアクション**: メール送信・カレンダー追加のみ

#### fin_s スキル（セッション終了）

```
1. 今日の作業内容を整理
2. 完了タスクをバックログで完了マーク
3. 新規発見タスクをバックログに追加
4. セッションログをsessions/YYYY-MM-DD.mdに保存
5. owner-profile.mdに学習内容を追記
6. 翌朝のための引き継ぎメモを作成
```

#### weekly_kpi スキル

```
入力: Manus AI週次レポート（ペースト）
↓
1. レポートをパース（LINE新規・累計・Application・Match・新規学生・新規企業）
2. kpi/db/weekly.md に追記
3. kpi/db/monthly.md の当月集計を更新
4. kpi/db/conversion.md の転換率を再計算
5. kpi/current.md のスナップショットを更新
6. strategy/sessions/YYYY-MM-DD.md にPDCAセッションを生成
```

## 6. KPI管理システム

### 6-1. 管理対象KPI（JVA/IB）

| KPI | 説明 |
|-----|------|
| LINE新規追加数 | 週ごとの新規LINE登録者数 |
| LINE累計 | LINE登録者の累計数 |
| Application数 | 週ごとの学生応募数 |
| Match数 | 成立したマッチング数 |
| LINE→App転換率 | Application ÷ LINE累計 |
| App→Match転換率 | Match ÷ Application |

### 6-2. KDBファイル構成

| ファイル | 内容 |
|---------|------|
| `kpi/db/weekly.md` | 週次実績テーブル（W01〜） |
| `kpi/db/monthly.md` | 月次集計・達成率 |
| `kpi/db/conversion.md` | 転換率の推移 |
| `kpi/current.md` | 最新スナップショット（いつでも参照できる状態） |
| `config/kpi-settings.md` | 目標値（月次目標・週次ペース） |

## 10. Notion AIスキルとして再現すべきスキル

Notionのパーソナライズ機能 + AIスキルを使って以下を設定する。

### コンテキスト登録（パーソナライズ）

Notion AIに以下のDBを常時参照させる:
- タスクDB（Q1から優先表示）
- プロジェクトDB
- セッションログDB（直近3件）
- KPI DB（直近4週）

### AIスキル定義

| スキル名 | コマンド | やること |
|---------|---------|---------|
| **朝ブリーフィング** | 「おはよう」 | カレンダー確認 + Q1タスク提示 + 未返信メール + 今日のMust3件 |
| **夜の締め** | 「お疲れ」「今日終わり」 | 今日の完了確認 + バックログ更新 + 明日の準備 |
| **タスク追加** | 「〇〇追加して」 | 象限自動判定してタスクDBに追加 |
| **進捗確認** | 「今日どう？」「何残ってる？」 | Q1タスクの残りを一覧表示 |
| **壁打ち** | 「相談したい」 | コンテキスト読み込み → 構造化して一緒に考える |
| **週次レビュー** | 「週次レビューして」 | 今週の成果・未完了・来週の優先事項をまとめる |

## 11. 重要ルール・制約

1. **メール送信・カレンダー追加は必ず確認を取る**（これ以外は自律実行OK）
2. **タスクの正データはNotionをマスターにする**（ローカルのbacklog.mdは廃止方向）
3. **セッションログは必ずNotionに書き残す**（Claude CodeもBase44も同じ場所に）
4. **APIキー・シークレットはチャットに出力しない**
5. **会話の冒頭は「前回の続き」から入る**（「はじめまして」禁止）
6. **回答フォーマット**: 結論→根拠→打ち手の順。表・箇条書き優先。
7. **推奨を先に出す**: 選択肢の羅列より「これが良い」を先に言う
