## 🔄 タスク自動検知ルール（アンビエント・常時有効）

> **このルールはすべてのセッションで常に適用される。スキル実行中以外でも有効。**

会話中にオーナーがタスクについて言及したら、**確認を取らずに即座にバックログを更新**する。

### 検知パターンと対応

| オーナーの発言例 | 対応 |
|---------------|------|
| 「〇〇やった」「〇〇完了した」「〇〇できた」「〇〇終わった」 | 該当タスクを `[x]` にマーク |
| 「〇〇もうやらなくていい」「〇〇なくなった」「〇〇キャンセル」「〇〇はいい」 | 該当タスクをバックログから削除 |
| 「〇〇やりたい」「〇〇追加して」「〇〇もやる必要がある」「〇〇しなきゃ」 | 適切な象限にタスクを追加 |
| 「〇〇は来週でいい」「〇〇は急がない」「〇〇は緊急じゃない」 | 象限を Q1→Q2 などに変更 |

### 更新対象ファイル（タスクの所属に応じて選択）

- JVA業務タスク → `company_jva/secretary/backlog.md`
- 個人・横断タスク → `company_personal/tasks/backlog.md`
- 両方に記載あり → 両方更新

### 更新後の通知

バックログを更新したら、会話の流れを妨げず一言だけ添える:
```
（バックログ更新済み: [タスク名] を[完了/削除/追加]）
```
