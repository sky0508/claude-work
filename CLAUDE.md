# グローバル設定 — Claude Code

## 基本ルール

- **言語**: 常に日本語で応答する
- **コンテキストの置き場所**: プロジェクト情報・作業ログはすべてNotionに置く。ローカルには置かない。
- **スタイル**: 簡潔・直接的。不要な前置きや要約はしない。

---

## コンテキスト管理

### アクティブコンテキストページ（Notionに一元管理）

作業状態の保存・復元先：

```
resumeページ: https://www.notion.so/Active-Context-Claude-Code-327f995fe92a81a7833be53abf1b1d29
```

セッション中の作業状態は以下の優先順位で管理する：

1. Notionページが渡された場合 → そのページがアクティブコンテキストページ。進捗・中間アウトプット・タスクリストはそこに書く
2. 渡されていない場合 → 上記のresumeページを使う

保存時は「追記」ではなく「## 作業状態」セクションを**全面書き直し**にする（古い情報と新しい情報の混在を防ぐ）。

---

## MEMORY.md ルール（コンテキスト汚染防止）

### MEMORY.md に書いていい情報

- 技術スタック、API仕様（横断的に使う技術情報のみ）
- どのMCPでどのワークスペースに接続するか
- ユーザーの永続的な好み・フィードバック

### MEMORY.md に書いてはいけない情報

- プロジェクト固有のDB ID、スクリプトURL、案件固有の要件
- 短命なタスクメモ、進捗情報
- セッション固有の作業状態（→ Notionのアクティブコンテキストページへ）

MEMORY.mdへの書き込みは必ずユーザーに確認を取ること。無許可で書き込まない。
情報には鮮度管理タグ `確認: YYYY-MM-DD` を付ける。

---

## スキル

- `save`: 「保存して」で発動 → 現在の作業状態をNotionのアクティブコンテキストページに全面書き直し
- `resume`: 「続き」で発動 → Notionのページを読み込んで作業再開
