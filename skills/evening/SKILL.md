# /evening — 夕方締めスキル v1

## 概要

1日の終わりに **タスク進捗を記録・バックログを更新** し、翌朝の `/morning` が正確なタスクリストを出せるようにする。
**会話やセッションを自動参照して質問を最小化する。ヒアリングは最大3問。**

---

## 実行フロー

### Step 1: 今日の状況を自動読み込み（サイレント実行）

以下を順番に読み込む。ユーザーへの報告は不要。

```
tasks/today.md                              → 今日のMust/Should/定期タスク一覧
company_jva/secretary/todos/YYYY-MM-DD.md  → JVA日次TODO（あれば）
company_jva/secretary/sessions/            → 今日作成されたセッションログ（あれば）
company_personal/secretary/sessions/       → 今日の個人セッションログ（あれば）
company_jva/secretary/backlog.md           → JVAバックログ現状
company_personal/tasks/backlog.md          → 個人バックログ現状
```

---

### Step 2: 完了タスクの自動検出

セッションログ・today.md のチェックボックスから **完了済みのタスクを自動抽出**する。

判定基準:
- `[x]` になっているタスク → 完了確定
- セッションログの「完了タスク」セクションに記載あり → 完了確定
- 会話の文脈で「完了した」「やった」と発言されているもの → 完了確定

---

### Step 3: 未完了タスクへのヒアリング（最大3問）

Step 2 で **完了が確認できなかった Must タスク** についてのみ質問する。

**質問ルール:**
- 1タスクにつき1文の短い質問のみ
- 「できましたか？」「進捗は？」「明日に持ち越しますか？」の3択で聞く
- 既にセッションで言及されているタスクは質問しない
- 最大3問で打ち切る（それ以上は翌朝に引き継ぎ）

**質問フォーマット:**
```
今日の進捗を確認させてください（最大3問）

Q1. [タスク名] — 完了 / 一部完了（続きあり） / 持ち越し（未着手）
Q2. [タスク名] — 完了 / 一部完了（続きあり） / 持ち越し（未着手）
Q3. [タスク名] — 完了 / 一部完了（続きあり） / 持ち越し（未着手）
```

---

### Step 4: バックログ更新（Automatic）

ヒアリング結果 + 自動検出結果をもとに **両バックログを更新**する。

**更新ルール:**

| 進捗 | バックログへの反映 |
|------|----------------|
| 完了 | `[ ]` → `[x]` に変更。完了済みセクションに移動 |
| 一部完了 | タスク名に `（進行中）` を追記 |
| 持ち越し | そのまま残す（期限がある場合は+1日延長を検討） |
| 新規タスク判明 | 適切な象限に追加 |
| 削除・不要になった | バックログから削除 |

---

### Step 5: JVA日次TODOに夕方ログを追記（Automatic）

`company_jva/secretary/todos/YYYY-MM-DD.md` が存在する場合、末尾に追記する:

```markdown
---
## 🌙 夕方進捗ログ（/evening YYYY-MM-DD）

### 完了
- [x] [タスク名]
...（なければ「なし」）

### 持ち越し（明日へ）
- [ ] [タスク名]
...（なければ「なし」）

### 備考
[ヒアリングで判明した追加情報があれば記載]
```

---

```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 他スキルとの連携

| 場面 | 使うスキル |
|------|----------|
| 朝のブリーフィング | `/morning` |
| 日中のタスク追加・相談 | `/secretary` |
| 夜の締め・進捗記録 | `/evening`（このスキル）|
| コード/深い作業のセッション終了 | `/fin_s` |

> `/evening` が更新したバックログを、翌朝の `/morning` が読み込む。この順番でタスクが最新化される。

---

## 重要ルール

1. **ヒアリングは最大3問** — 疲れた状態で多く聞かない
2. **セッション参照を優先** — 会話で判明している分は質問しない
3. **バックログ更新は確認不要** — 即時更新してOK
4. **JVA todos ファイルへの追記のみ** — 上書き禁止
5. **tasks/today.md は更新しない** — 翌朝の /morning が上書きするため
6. **明日の最優先は1〜2件のみ** — 多すぎると機能しない
