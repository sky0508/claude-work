# Manus スライドアウトライン生成スキル

**重要: アウトラインはユーザーが「OK」「それで行こう」と言うまで出力しない。**
対話を通じて論点を深め、何を伝えるべきかを一緒に整理してからアウトラインにする。

**Manusアセット:** `~/claude-workspace/company_jva/output/manus/`

---

## フロー

```
① 関連ファイルを参照（サイレントに）
        ↓
② 目的・状況を整理して提示
        ↓
③ 「なぜそうなっているのか」「何が一番重要か」を対話で深ぼる
   ← ここに時間をかける。ユーザーの思考を引き出す。
        ↓
④ 論点が整ったら「こういう構成はどうですか？」と提案
        ↓
⑤ ユーザーが「OK」「それで行こう」と言ったらアウトライン出力
        ↓
⑥ アウトラインOK後 → Manusプロンプト出力
```

---

## ステップ1: 用途の特定 + ファイル参照

ユーザーの発言から用途を判定し、対応するファイルをサイレントに読む。

| 用途 | 参照ファイル（必須） |
|------|-----------------|
| A. 週次ミーティング | kpi/current.md / kpi/db/weekly.md / pm/projects/ib-overall.md / secretary/sessions/（最新1〜2件） / secretary/todos/（最新） |
| B. 外部営業・新規企業 | output/jva-ib-guide.md / kpi/current.md / research/topics/student-persona.md |
| C. 学生集客・説明 | output/jva-ib-guide.md / research/topics/student-persona.md / kpi/current.md |
| D. 既存クライアント交渉 | sales/clients/[対象].md / research/topics/matching-bottleneck-analysis.md / research/topics/student-persona.md |
| E. 内部KPIレポート | kpi/current.md / kpi/db/weekly.md / kpi/db/conversion.md |
| F. 戦略・意思決定 | strategy/briefs/（関連） / pm/projects/ib-overall.md / strategy/insights/（最新） |

## ステップ2: 目的・状況の提示

読んだ内容をもとに、以下の形式で状況を提示する。

```
【読んだファイルから見えていること】
- 数値・事実: [重要な数値、変化、異常値]
- 前回からの変化: [進んだこと、止まっていること]
- 気になっている点: [自分が「なぜ？」と思ったこと]

【このスライドの目的について確認】
→ [目的が明確な場合は確認不要。不明な場合のみ質問]
```

---

## ステップ3: 対話で深ぼる ← 最も重要なステップ

「なぜそうなっているのか」「本当に伝えたいことは何か」をユーザーと一緒に考える。

**進め方:**
- 自分の仮説・見解を先に言う（「〜だと思うんですが、どう思いますか？」）
- 1回に1〜2個の問いに絞る（質問を並べすぎない）
- ユーザーの答えを受けてさらに深める
- 「それはなぜですか？」「他に考えられる要因は？」「優先するとしたら？」

**対話を続けるべきサイン:**
- まだ「なぜ」の答えが出ていない
- 「何を優先するか」が決まっていない
- スライドの核心メッセージが1行で言えない

**アウトラインに移るサイン:**
- コアメッセージが明確になった
- 伝えるべき論点が3〜5点に絞れた
- ユーザーが「そんな感じで」「それでいい」と言った

---

## ステップ4: 構成案を提示

対話が終わったら構成案を提示する。

```
## こういう構成にしようと思います

コアメッセージ: [1行]
構成の方向性: [例: 問題提起 → 原因 → 打ち手 → 意思決定]

| # | タイトル | 内容の要点 |
|---|---------|----------|
| 1 | Cover   | ...      |
| 2 | ...     | ...      |

これでアウトラインを作りますか？
```

ユーザーのOKを待つ。修正があれば反映してから再確認。

---

## ステップ5: アウトライン出力

OKが出たら詳細なアウトラインを出力する。

各スライドに:
- Layout hint（Manusへのレイアウト指定）
- Content（箇条書き・数値）
- Key message（1行）

---

## ステップ6: アクションと優先順位の確認 ← 秘書連携ポイント

アウトライン出力後、議論の中で出てきたアクション候補を整理して確認する。

```
## 議論の中で出てきたアクション候補

以下をTODOに追加しますか？

| # | アクション | 緊急度 | 重要度 | 提案優先度 |
|---|-----------|--------|--------|-----------|
| 1 | [アクション1] | 高/中/低 | 高/中/低 | Q1/Q2/Q3 |
| 2 | [アクション2] | ... | ... | ... |

優先順位について確認させてください：
→ [例: 「#1と#2、来週中に両方やりますか？それとも#1を先に片付けますか？」]

TODOに追加する場合は「秘書を呼んでください」と言っていただければ、
秘書がTODOファイルと進捗管理を引き継ぎます。
```

**判断の軸（優先順位の提案根拠）:**
- 即効性があるか（今週中に結果が出るか）
- ボトルネック解消に直結するか
- 2名リソースで現実的にできるか

---

## ステップ7: Manusプロンプト出力

アウトラインOK後、コピペ用プロンプトとして出力する。

**ブランド指定（短縮版 / Manus学習済みのため簡潔に）:**
```
=== JVA BRAND ===
Colors: Primary Red #BE1229 / Purple #6251D0 / Header Dark #1A0508 / Off-White bg #FAF5F5
Fonts: Montserrat Bold (headings) / Lato (body)
RAG: Green #27AE60 / Yellow #F59E0B / Red #BE1229
Style: Minimal. Dark header bar + red line. One message per slide. No animations.
=== END ===
```

出力フォーマット:
```
[ブランド短縮版]

Create a [N]-slide [用途] in [English/Japanese].
Date: [日付] | Audience: [対象] | Goal: [目的]

--- SLIDE OUTLINE ---
Slide 1 — Cover: Title / Subtitle / Catchphrase
Slide N — [タイプ] | "[タイトル]"
  Layout: [hint]
  Content: [箇条書き]
  Key message: [1行]
--- END ---

Speaker notes: required. Export: .pptx
```
